# 887Assistant

Многофункциональный Telegram-бот:

- 📁 **Хранение файлов** — загрузка, список, скачивание, переименование, удаление, поиск.
- 📅 **Календарь** — события по дням + синхронизация с телефоном через `.ics`/`webcal`.
- 🤖 **ИИ-чат** — общение с нейросетью (Kimi от Moonshot по умолчанию, с бесплатными альтернативами).
- 🎮 **FACEIT ники** — проверка доступности никнеймов (одиночная/массовая, все 3-символьные).

> Архитектура модульная: каждый блок — отдельный пакет в `app/modules/*` с собственным
> aiogram-роутером и сервис-слоем. Общий контракт данных — в `app/db/models.py`.

## Возможности

- **Файлы:** приём любых файлов, список с пагинацией, скачивание, переименование, удаление,
  поиск по имени; доступ только к своим файлам.
- **Календарь:** месячная сетка с отметкой дней с событиями, просмотр дня, добавление событий
  (название/описание/время/напоминание), напоминания (APScheduler), личная `.ics`/`webcal`-лента
  для подписки на телефоне.
- **ИИ-чат:** диалог с сохранением контекста, провайдер по умолчанию Kimi (Moonshot) с
  переключаемыми бесплатными альтернативами (Gemini, Groq, OpenRouter) и сбросом диалога.
- **FACEIT:** проверка одного/нескольких ников и массовый перебор всех 3-символьных ников
  (отдельно буквы и цифры) с прогрессом, остановкой и копируемым списком свободных.

## Быстрый старт

```bash
git clone <repo> && cd 887assistant
cp .env.example .env          # заполните TELEGRAM_BOT_TOKEN и остальные ключи
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python -m app.main
```

Или через Docker:

```bash
cp .env.example .env          # заполните ключи
docker compose up --build
```

## Переменные окружения

См. `.env.example`. Ключевые:

| Переменная | Назначение |
|---|---|
| `TELEGRAM_BOT_TOKEN` | токен бота от [@BotFather](https://t.me/BotFather) |
| `DATABASE_URL` | SQLite (по умолчанию) или PostgreSQL. Railway отдаёт `postgresql://…` — приложение само нормализует его в `postgresql+asyncpg://…` |
| `STORAGE_BACKEND` | `local` (по умолчанию) или `s3` — куда складывать файлы |
| `S3_ENDPOINT_URL` / `S3_ACCESS_KEY_ID` / `S3_SECRET_ACCESS_KEY` / `S3_BUCKET` / `S3_REGION` | параметры объектного хранилища (Cloudflare R2 / Storj / S3) |
| `S3_FORCE_PATH_STYLE` | `true` для провайдеров, которым нужен path-style (например Storj); по умолчанию `false` |
| `FILE_STORAGE_PATH` | каталог для файлов при `STORAGE_BACKEND=local` |
| `LLM_PROVIDER` / `LLM_API_KEY` / `LLM_MODEL` | провайдер и ключ ИИ-чата |
| `WEB_SEARCH_PROVIDER` / `TAVILY_API_KEY` | веб-поиск для ИИ (по умолчанию `tavily`) |
| `FACEIT_API_KEY` | ключ FACEIT Data API ([developers.faceit.com](https://developers.faceit.com)) |
| `BASE_URL` | публичный URL для подписки на календарь (`.ics`/`webcal`) |

### Где взять ключи

- **Telegram:** [@BotFather](https://t.me/BotFather) → `/newbot`.
- **FACEIT:** [developers.faceit.com](https://developers.faceit.com) → App Studio → API Keys → **Server side**.
- **Kimi / Moonshot:** платформа Moonshot AI → API keys (`https://api.moonshot.ai/v1`).
- **Бесплатные альтернативы ИИ:** Google Gemini (AI Studio), Groq, OpenRouter (free-модели).
- **Веб-поиск (Tavily):** [tavily.com](https://tavily.com) → API key (бесплатный тариф). DuckDuckGo на хостинге ненадёжен (таймауты с дата-центровых IP), поэтому по умолчанию используется Tavily.
- **Cloudflare R2:** dash.cloudflare.com → R2 → создать bucket и API-токен (S3-совместимый, 10 ГБ бесплатно).

## Развёртывание на Railway (Postgres + Cloudflare R2)

Файловая система контейнера Railway **эфемерная** — поэтому БД и файлы нужно держать снаружи, иначе они стираются при каждом деплое.

1. **БД — Railway Postgres:** добавьте плагин Postgres; Railway пробросит `DATABASE_URL=postgresql://…` (нормализуется автоматически).
2. **Файлы — Cloudflare R2 (10 ГБ бесплатно):** задайте переменные:
   ```bash
   STORAGE_BACKEND=s3
   S3_ENDPOINT_URL=https://<account_id>.r2.cloudflarestorage.com
   S3_ACCESS_KEY_ID=<r2-access-key-id>
   S3_SECRET_ACCESS_KEY=<r2-secret-access-key>
   S3_BUCKET=<bucket>
   S3_REGION=auto
   ```
3. **Веб-поиск ИИ:** `WEB_SEARCH_PROVIDER=tavily` + `TAVILY_API_KEY=...`.
4. **Веб-сервер:** `WEB_PORT=${{PORT}}`, `WEB_HOST=0.0.0.0`, `BASE_URL=https://<домен>.up.railway.app`.

> Volume на тарифе Hobby ограничен 5 ГБ, поэтому для «10 ГБ» под файлы используется именно объектное хранилище (R2). Маленький Postgres под метаданные с запасом укладывается в лимиты.

### Storj (EU)

Storj предоставляет S3-совместимый шлюз. Для хранения данных в ЕС создайте access/secret в консоли Storj на **EU1-сателлите** (data residency для ЕС) и используйте path-style адресацию:

```bash
STORAGE_BACKEND=s3
S3_ENDPOINT_URL=https://gateway.storjshare.io
S3_REGION=eu1
S3_ACCESS_KEY_ID=<storj-access-key>
S3_SECRET_ACCESS_KEY=<storj-secret-key>
S3_BUCKET=<bucket>
S3_FORCE_PATH_STYLE=true
```

## Модули

### 📁 Файлы

Отправьте боту любой файл — документ, фото, видео, аудио или голосовое сообщение — и он
сохранит его. Маленькие файлы (до `FILE_MAX_DOWNLOAD_BYTES`, по умолчанию ~20 МБ — лимит
Telegram на скачивание) загружаются на сервер, крупные хранятся по ссылке Telegram
(их всё равно можно переслать обратно). В разделе «📥 Мои файлы» доступен список с
постраничной навигацией, а по кнопке файла открывается карточка с размером, типом и датой.
Любой файл можно скачать (бот пришлёт его заново по `telegram_file_id`), переименовать или
удалить. Кнопка «🔎 Поиск» находит файл по части имени без учёта регистра. Каждый
пользователь видит только свои файлы.

### 📅 Календарь и синхронизация

Модуль хранит личные события пользователя и показывает их прямо в Telegram, а для
телефона отдаёт персональную ленту в формате `.ics`/`webcal`.

**Открытие календаря.** Кнопка «📅 Календарь» в главном меню рисует сетку текущего месяца
как inline-клавиатуру: шапка с названием месяца и годом, кнопки навигации «◀»/«▶»,
строка дней недели и сетка 7×N с номерами дней. Дни, в которых есть события, помечаются
точкой (например `•15`). Внизу — кнопки «📲 Синхронизация», «Сегодня» и «⬅️ В меню».
Время везде показывается в часовом поясе пользователя (`User.timezone`, по умолчанию `UTC`),
а в базе всё хранится в UTC.

**Просмотр дня.** Нажатие на день открывает список его событий: для каждого показывается
время начала в вашем поясе и название (для событий на весь день — «весь день»). Доступны
кнопки на каждое событие, «➕ Добавить событие» и «◀ Месяц». Карточка события содержит
детали и кнопку «🗑 Удалить».

**Добавление события.** Запускается из дня кнопкой «➕ Добавить событие» и идёт пошагово
(FSM): дата уже подставлена из выбранного дня → **название** → **описание** (можно отправить
`-`, чтобы пропустить) → **время** в формате `ЧЧ:ММ` (или слово `весь день` для события на
весь день) → выбор **напоминания** кнопками (нет / 10м / 30м / 1ч / 1 день). Дата и время
разбираются в вашем часовом поясе и конвертируются в UTC при сохранении.

**Напоминания.** Если для события выбрано напоминание, фоновая задача (APScheduler,
интервал 60 секунд) проверяет события, у которых окно напоминания открыто
(`старт − N минут ≤ сейчас ≤ старт`), отправляет владельцу сообщение в Telegram и помечает
напоминание отправленным, чтобы оно не повторялось. В файле `.ics` напоминание выгружается
как `VALARM` с триггером `-PTxxM`.

**Синхронизация с телефоном (`.ics` / `webcal`).** Кнопка «📲 Синхронизация» показывает две
ссылки на вашу персональную ленту (для копирования):

```
https://<BASE_URL>/calendar/<calendar_token>.ics
webcal://<host>/calendar/<calendar_token>.ics
```

Как подписаться:

- **Google Календарь:** «Другие календари → Подписаться по URL» → вставьте ссылку.
- **Apple/iOS:** «Настройки → Календарь → Учётные записи → Добавить учётную запись →
  Другое → Подписной календарь».

⚠️ **Важно про ограничение.** В Telegram нет встроенного календаря, поэтому бот хранит события
у себя и отдаёт их через персональную ленту `.ics`/`webcal`. Подписка в Google/Apple Calendar
даёт **одностороннюю синхронизацию только для чтения**: события появляются на телефоне, но
изменить или удалить их в телефонном календаре нельзя — это делается в боте. Кроме того,
телефон обновляет подписные календари не мгновенно (интервал может составлять несколько часов).

**Возможное расширение (не реализовано).** Полноценная двусторонняя синхронизация потребовала бы
Google Calendar API (OAuth2) или CalDAV. Поля для Google OAuth уже зарезервированы в конфиге
(`GOOGLE_OAUTH_*`), но интеграция в этом модуле не реализована.

### 🤖 ИИ-чат

Бот умеет общаться с нейросетью. Откройте «🤖 ИИ-чат» в меню — после этого просто пишите
сообщения текстом, и ассистент будет отвечать (история диалога сохраняется на пользователя).
Кнопка «🧹 Сбросить диалог» очищает историю.

**Провайдер по умолчанию: Kimi через Moonshot** (`kimi-k2.6`). API Moonshot
OpenAI-совместим (`https://api.moonshot.ai/v1`).

Как получить ключ Moonshot: зарегистрируйтесь на платформе Moonshot AI
(`https://platform.moonshot.ai`), в разделе API Keys создайте ключ и пропишите в `.env`:

```bash
LLM_PROVIDER=moonshot
LLM_API_KEY=sk-...ваш-ключ...
# при желании сменить модель, например на Kimi 2.7 (для кода):
# LLM_MODEL=kimi-k2.7-code
```

Доступна и более новая **Kimi 2.7 (`kimi-k2.7-code`)** — укажите её в `LLM_MODEL`. Полный
список моделей вашего ключа — через эндпоинт `/models`.

**Бесплатные альтернативы** (переключение через `LLM_PROVIDER` / `LLM_API_KEY` / `LLM_MODEL`):

| `LLM_PROVIDER` | Base URL | Модель по умолчанию | Где взять ключ |
|---|---|---|---|
| `moonshot` (по умолчанию) | `https://api.moonshot.ai/v1` | `kimi-k2.6` | platform.moonshot.ai |
| `gemini` | `https://generativelanguage.googleapis.com/v1beta/openai` | `gemini-2.0-flash` | aistudio.google.com/apikey |
| `groq` | `https://api.groq.com/openai/v1` | `llama-3.3-70b-versatile` | console.groq.com/keys |
| `openrouter` | `https://openrouter.ai/api/v1` | `deepseek/deepseek-chat` | openrouter.ai/keys |
| `openai_compatible` | из `LLM_BASE_URL` | из `LLM_MODEL` | любой OpenAI-совместимый сервис |

Бесплатные тарифы Gemini / Groq / OpenRouter имеют свои лимиты по запросам в минуту/день —
смотрите актуальные ограничения у провайдера. `LLM_MAX_HISTORY_MESSAGES` (по умолчанию 20)
ограничивает длину хранимой истории, `LLM_REQUEST_TIMEOUT` (60) — таймаут запроса.
Если ключ не задан, бот вежливо попросит установить `LLM_API_KEY`.

### 🎮 FACEIT ники

Модуль проверяет доступность никнеймов через официальный **FACEIT Data API**
(`GET /data/v4/players?nickname=<ник>`). Нужен server-side ключ `FACEIT_API_KEY`
(developers.faceit.com → App Studio → API Keys). Без ключа доступен только справочный
раздел «Про смену ника / IDLE».

**Два режима:**

- **✍️ Проверить ники** — пришлите один или несколько ников через пробел или с новой строки
  (до 50 за раз). Бот вернёт по строке на ник: ✅ свободен, ⛔ занят, ⚠️ ошибка. Свободные
  ники приходят в `<code>…</code>` — тап по нику копирует его.
- **🔤 Все 3-буквенные (a–z)** / **🔢 Все 3-значные (0–9)** — массовый перебор всего
  пространства: **17576** буквенных и **1000** цифровых комбинаций. Перед стартом
  показывается предупреждение об объёме и времени при текущем лимите запросов, есть
  подтверждение «▶️ Запустить» и кнопка «⏹ Остановить» с живым прогрессом
  (сделано/всего, найдено свободных). При остановке возвращается всё, что успели проверить.

**Точность.** API достоверно различает только **занят / свободен** (HTTP 200 → занят,
404 → свободен). 429 — лимит (бот делает паузу и повтор), прочие/сетевые ошибки —
«неизвестно» (свободным НЕ считаем). Поиск **чувствителен к регистру**: `Nick` и `nick` —
разные ники. Запросы намеренно ограничены по скорости (`FACEIT_RATE_LIMIT_RPS`) и
кэшируются (`FACEIT_CACHE_TTL`) — без «долбёжки» API, в рамках ToS FACEIT.

**Смена ника и «IDLE»-захват (правила FACEIT):**

- **Своя смена:** Premium — 1 бесплатная смена раз в 3 месяца; иначе предмет
  «Nickname Change» в FACEIT Shop.
- **Idle Nickname Claim:** ник можно запросить, только если владелец отвечает ВСЕМ условиям:
  неактивен более 12 месяцев и без долгосрочных банов; нет активной подписки; не verified;
  не удалён/деактивирован; ник не оскорбительный и не занят в вариантах с другим регистром.
- Неактивным шлют письмо за 7 дней; любой вход сбрасывает 12-месячный таймер. При захвате
  прежний владелец получает обезличенный ник (например `yellow1234`), 1000 FACEIT Points и
  одну бесплатную смену. Захват — перк Premium либо ~$4.99 / 5000 FACEIT Points, через
  Shop / Account Settings (НЕ при обычной регистрации). Ники **удалённых** аккаунтов не
  освобождаются никогда.

> ⚠️ **Честно о точности:** определить, является ли занятый ник «idle-claimable», только по
> этому эндпоинту нельзя — это лишь эвристическая оценка по последней активности, а не
> гарантия. API отвечает только «занят/свободен».
